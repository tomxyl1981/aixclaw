## 三、核心技术：语义记忆与数据主权

### 3.1 为什么语义记忆是核心竞争力？

**传统 AI 的局限**:
- 没有长期记忆，每次对话从零开始
- 不理解用户业务逻辑
- 数据存储在云端，用户失去控制

**AIX Box 的突破**:
```
用户输入 (对话/文档)
     ↓
Ontology 语义理解 (业务逻辑)
     ↓
Knowledge Graph 结构化存储 (实体-关系-属性)
     ↓
Memory + Embedding 长期记忆
     ↓
Wallet DID 身份绑定 (数据主权)
     ↓
AI Agent 个性化服务
```

### 3.2 三层记忆架构

| 层级 | 功能 | 技术 |
|-----|------|------|
| **Ontology** | 定义业务概念与规则 | JSON-LD |
| **Knowledge Graph** | 存储结构化知识 (实体-关系) | SQLite + RDF |
| **Memory** | 原始数据与向量索引 | Local DB + Chroma |

**示例**: 园区企业关系
```
记忆层: "昨天见了李总，他公司做 AI 芯片"
     ↓
知识图谱: 实体[李总] --[是]--> 实体[AI芯片公司]
     ↓
Ontology: 人物 → 就职于 → 企业 (类型检查)
```

### 3.3 钱包即身份 (Wallet = DID)

**每个 AIX Box 内置硬件钱包**:
- 钱包地址 = 去中心化身份 (DID)
- 私钥保护 = ATECC608B 硬件芯片
- 助记词恢复 = 跨设备身份同步

**数据主权保障**:
```
所有知识图谱实体绑定 ownerDID
     ↓
数据修改需身份签名 (防篡改)
     ↓
用户控制访问授权
     ↓
随时导出/迁移数据
```

### 3.4 与 Anthropic 的本质差异

| 维度 | Anthropic (中心化) | AIX Box (去中心化) |
|-----|-------------------|-------------------|
| 记忆存储 | 云端服务器 | 本地硬件 + 可选 BPFS |
| 语义理解 | 黑盒模型 | 可解释 Ontology |
| 数据主权 | 服务商掌控 | 用户掌控 |
| 身份绑定 | 无 | 钱包 DID |
| 跨平台 | 锁定 | 开放标准 |
| 离线使用 | 不支持 | 完全支持 |

## 四、商业闭环：数据 → Skill → 收益

### 4.1 用户价值积累路径

```
用户使用 AIX Box
     ↓
积累语义记忆 (Ontology + Knowledge Graph)
     ↓
打包成 Skill (工作流 + 知识)
     ↓
上架 Skill Store
     ↓
其他用户购买使用
     ↓
创作者获得收益 (AIX/Coin Hour)
```

### 4.2 示例场景

**园区企业主**:
1. 使用 Box 管理客户关系
2. 积累行业知识图谱
3. 打包成"客户关系管理 Skill"
4. 出售给其他园区企业
5. 持续获得被动收入

**律师**:
1. 使用 Box 处理合同
2. 积累法律案例与模板
3. 打包成"合同审查助手 Skill"
4. 按次收费

### 4.3 Skill Store 生态

```
创作者端:
- 使用 Box 钱包签名发布 Skill
- DID 绑定创作者身份
- 收益自动转入钱包

购买者端:
- 验证 Skill 签名 (防篡改)
- 查看创作者 DID 历史 (信誉)
- 使用 Skill 积累新记忆

平台端:
- 20% 平台抽成
- 80% 归创作者
- 基于 DID 的声誉系统
```

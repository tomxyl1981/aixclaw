# 中俄双向贸易流程
## 从订单开始，完整流程梳理

---

## 🔄 双向贸易场景

**场景A：** 俄罗斯买家 → 中国卖家（中国商品出口俄罗斯）
**场景B：** 中国买家 → 俄罗斯卖家（俄罗斯商品进口中国）

---

## 📋 场景A：俄罗斯买家购买中国商品

### 第一步：下单

```
俄罗斯买家
    │
    │ 1. 打开AIX钱包APP
    │ 2. 浏览商品（或输入商品链接）
    │ 3. 选择商品，点击"下单"
    │ 4. 填写收货地址
    │ 5. 选择支付方式（Coin Hour / AIX）
    │ 6. 确认订单
    ↓
订单创建成功
```

**技术细节：**
- 买家AIX Box运行钱包客户端
- 通过LBRTYnet发送订单消息
- 订单格式：JSON + 数字签名

**订单数据：**
```json
{
  "order_id": "RU-CN-20260409-001",
  "buyer": {
    "address": "俄罗斯买家钱包地址",
    "shipping": "莫斯科 XXX街XX号",
    "phone": "+7-XXX-XXX-XXXX"
  },
  "seller": {
    "address": "中国卖家钱包地址"
  },
  "items": [
    {
      "product_id": "CN-001",
      "name": "电子产品配件",
      "quantity": 100,
      "price": 10000,  // Coin Hour
      "hs_code": "8542.31"
    }
  ],
  "total": 10000,  // CH
  "currency": "CH",
  "timestamp": "2026-04-09T01:00:00Z",
  "signature": "买家签名"
}
```

---

### 第二步：托管锁定

```
买家确认订单
    │
    │ 1. 钱包提示"锁定Coin Hour到托管合约"
    │ 2. 买家确认锁定
    │ 3. Coin Hour转入托管地址
    ↓
托管合约创建
```

**托管合约：**
```
托管地址 = 2-of-3 多签名
├── 买家公钥
├── 卖家公钥
└── 仲裁节点公钥

释放条件：
- 买家 + 卖家签名 → 释放给卖家
- 买家 + 仲裁签名 → 释放给卖家（超时）
- 卖家 + 仲裁签名 → 退回买家（争议）
```

**买家看到：**
```
✅ 订单已创建
💰 10,000 CH 已锁定托管
⏳ 等待卖家确认...
```

---

### 第三步：卖家确认

```
订单消息通过LBRTYnet到达中国
    │
    │ 1. 卖家AIX Box收到订单通知
    │ 2. 卖家查看订单详情
    │ 3. 卖家确认有货
    │ 4. 点击"接单"
    ↓
订单状态：已确认
```

**卖家看到：**
```
📦 新订单
买家：俄罗斯莫斯科
商品：电子产品配件 x 100
金额：10,000 CH
状态：待确认

[接单] [拒绝]
```

**点击接单后：**
```

📦 准备发货中...
```

---

### 第四步：备货发货

```
卖家确认订单
    │
    │ 1. 准备货物
    │ 2. 打包
    │ 3. 打印物流标签
    │ 4. 扫描货物二维码
    │ 5. 系统自动生成物流单号
    ↓
货物已发出
```

```
    │ 5. 合规检查
    ↓
清关完成
```

## 🔄 双向流程对比

| 阶段 | 俄→中（买中国商品） | 中→俄（买俄罗斯商品） |
|------|-------------------|---------------------|
| 下单 | 俄罗斯买家 | 中国买家 |
| 支付 | CH托管锁定 | CH托管锁定 |
| 确认 | 中国卖家接单 | 俄罗斯卖家接单 |
| 发货 | 中国仓库 | 俄罗斯仓库 |
| 清关 | 出口：满洲里<br>进口：后贝加尔斯克 | 出口：后贝加尔斯克<br>进口：满洲里 |
| 物流 | 中国→俄罗斯 | 俄罗斯→中国 |
| 收货 | 俄罗斯买家确认 | 中国买家确认 |
| 结算 | CH释放给中国卖家 | CH释放给俄罗斯卖家 |

## 🛡️ 异常处理

### 情况1：卖家不发货

```
订单确认后，卖家超时未发货（72小时）
    │
    │ 1. 系统自动提醒卖家
    │ 2. 再过24小时仍未发货
    │ 3. 订单自动取消
    │ 4. Coin Hour退回买家
    ↓
交易取消
```

### 情况2：货物丢失

```
物流追踪显示停滞超过7天
    │
    │ 1. 买家发起争议
    │ 2. 仲裁节点介入
    │ 3. 查看物流记录
    │ 4. 裁决：退款或补偿
    ↓
争议解决
```

### 情况3：货物质量问题

```
买家收货后发现质量问题
    │
    │ 1. 买家发起争议
    │ 2. 上传证据（照片/视频）
    │ 3. 卖家回应
    │ 4. 仲裁节点裁决
    │ 5. 部分退款 / 全额退款 / 驳回
    ↓
争议解决
```

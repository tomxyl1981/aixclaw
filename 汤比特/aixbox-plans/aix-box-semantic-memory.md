## 二、Ontology 层设计

### 2.1 Ontology 定义格式 (JSON-LD)

```json
{
  "@context": {
    "aix": "https://aix.box/ontology/",
    "schema": "https://schema.org/"
  },
  "@type": "Ontology",
  "name": "电商领域本体",
  "version": "1.0",
  "classes": [
    {
      "@id": "aix:Customer",
      "@type": "Class",
      "label": "客户",
      "properties": [
        {"@id": "aix:name", "range": "xsd:string", "required": true},
        {"@id": "aix:email", "range": "xsd:string", "pattern": "^\\S+@\\S+\\.\\S+$"},
        {"@id": "aix:vipLevel", "range": ["普通", "银卡", "金卡", "钻石"]}
      ]
    }
  ],
  "relations": [
    {
      "@id": "aix:purchases",
      "domain": "aix:Customer",
      "range": "aix:Order",
      "cardinality": "1-to-many"
    }
  ]
}
```

### 2.2 Ontology 市场

- 预置 Ontology：电商、医疗、法律、通用
- 用户自定义：行业特定、企业内部
- Skill Store 交易：购买/出售 Ontology

## 四、数据主权保障

### 4.1 分层加密

```
Ontology 层 (公共):
├── 轻度加密/不加密
└── Skill Store 可交易

知识图谱层 (私有):
├── AES-256-GCM 加密
├── 密钥: 用户主密钥 (ATECC608B)
└── 用户可随时导出

记忆层 (完全私有):
├── 硬件级加密
├── 本地 NVMe 存储
└── 可选 BPFS 备份
```

### 4.2 用户控制权

| 权利 | 实现 |
|-----|-----|
| 导出 | 一键导出 JSON-LD |
| 删除 | 本地删除 + 撤销备份 |
| 迁移 | 跨设备/跨平台 |
| 共享 | 授权控制 |

## 九、钱包即身份：Box 内置身份层

### 9.1 核心设计

```
每个 AIX Box 内置硬件钱包
         │
         ├── 钱包地址 = 用户 DID (去中心化身份)
         ├── 私钥 = 身份验证凭证 (ATECC608B 保护)
         └── 助记词 = 身份恢复机制
```

**身份三层结构**:

```
┌─────────────────────────────────────────────────────┐
│              身份层 (Identity Layer)                 │
├─────────────────────────────────────────────────────┤
│                                                      │
│  DID (去中心化身份)                                   │
│  did:aix:0x742d35Cc6634C0532925a3b8D4C9db96590f6C7E │
│         │                                            │
│         ├── 钱包功能                                  │
│         │   ├── AIX 存储与转账                        │
│         │   ├── Coin Hour 管理                        │
│         │   └── 交易签名                              │
│         │                                            │
│         ├── 身份功能                                  │
│         │   ├── 知识图谱实体身份绑定                  │
│         │   ├── Skill 创作者身份验证                  │
│         │   └── 跨 Box 身份认证                       │
│         │                                            │
│         └── 数据主权功能                              │
│             ├── 加密密钥派生                          │
│             ├── 数据签名与验证                        │
│             └── 授权管理                              │
│                                                      │
└─────────────────────────────────────────────────────┘
```

### 9.2 钱包与知识图谱的绑定

```
知识图谱中的每个实体都与钱包身份关联

示例:
┌─────────────────────────────────────────────┐
│  实体: 张三 (Customer)                       │
│  ─────────────────────────────────          │
│  属性:                                       │
│    - name: "张三"                           │
│    - email: "zhangsan@example.com"          │
│    - vipLevel: "金卡"                        │
│                                              │
│  身份绑定:                                   │
│    - ownerDID: did:aix:0x742d...           │
│    - createdBy: did:aix:0x742d...          │
│    - signature: 0x9a8f7e... (防篡改)        │
│    - provenance: 可追溯的创建历史           │
└─────────────────────────────────────────────┘
```

**价值**:
- 数据可溯源：知道谁创建了这条数据
- 防篡改：任何修改都需要身份签名
- 权限控制：基于 DID 的细粒度访问控制

### 9.3 跨 Box 身份认证

```
Box A (办公室)                Box B (家里)
   │                              │
   │  相同 DID / 助记词恢复        │
   │  =========================>  │
   │                              │
   └── 身份同步                   └── 身份同步
        │                              │
        ▼                              ▼
   相同的知识图谱                  相同的知识图谱
```

**场景**:
- 办公室 Box 录入客户信息
- 回家用家里 Box，登录相同 DID

- 无缝延续工作

### 9.7 技术实现要点

```python
class BoxIdentity:
    """AIX Box 身份管理"""
    
    def __init__(self, wallet):
        self.wallet = wallet  # 硬件钱包 (ATECC608B)
        self.did = f"did:aix:{wallet.address}"
    
    def sign_data(self, data):
        """用钱包私钥签名数据"""
        return self.wallet.sign(data)
    
    def derive_key(self, purpose):
        """派生特定用途的密钥"""
        # 基于钱包主密钥派生
        return HKDF.derive(self.wallet.master_key, purpose)
    
    def create_entity(self, entity_data):
        """创建知识图谱实体，绑定身份"""
        entity = {
            **entity_data,
            "ownerDID": self.did,
            "createdAt": now(),
            "signature": self.sign_data(entity_data)
        }
        return entity
    
    def grant_access(self, grantee_did, resource, permissions):
        """授权给其他 DID"""
        authorization = {
            "grantor": self.did,
            "grantee": grantee_did,
            "resource": resource,
            "permissions": permissions,
            "signature": self.sign_data({...})
        }
        return authorization
```

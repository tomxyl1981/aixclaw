## 项目概述

OpenRelay 是一个聚合多家服务商免费AI模型的工具，实现"跑代码零消费"。

### 核心数据

| 指标 | 数值 |
|------|------|
| 聚合服务商 | 50+ 家 |
| 免费模型 | 580+ 个 |
| 连续运行测试 | 18小时 |
| 花费 | 0元 |

---

## 核心功能

### 1. 模型聚合
- 聚合50+服务商的免费额度
- 单一API访问多个模型
- 统一接口格式

### 2. 自动配额切换
- 模型组配置
- 额度耗尽自动切换下一个
- 无缝衔接不中断任务

### 3. 零成本运行
- 充分利用各平台免费额度
- 智能分配避免浪费
- 长时间任务零花费

## 技术架构推测

```
用户请求
    ↓
OpenRelay（聚合层）
    ↓
模型组选择（配额管理）
    ↓
服务商A → 服务商B → 服务商C...（自动切换）
    ↓
返回结果
```

### 关键组件

| 层 | 功能 |
|------|------|
| 聚合层 | 统一API，隐藏服务商差异 |
| 配额层 | 检测免费额度，自动切换 |
| 模型组 | 配置备选模型池 |
| 监控层 | 实时状态，错误恢复 |

## 建议集成方案

### 混合架构

```
AIX Box
├── 本地推理（大任务、敏感数据）
├── OpenRelay（轻量任务、免费优先）
└── Coin Hour结算（增值服务）
```

### 策略

| 任务类型 | 推理方式 |
|----------|----------|
| 敏感数据 | 本地模型（数据主权）|
| 长时间任务 | OpenRelay（零成本）|
| 高精度需求 | 付费模型（Coin Hour）|
| 日常对话 | OpenRelay免费模型 |

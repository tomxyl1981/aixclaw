## 二、技术原理

### 代码知识图谱
```
源代码 → 解析 → 知识图谱（本地构建）
         ↓
    函数调用关系
    类继承结构
    模块依赖图
    变量作用域
```

### 与传统代码检索对比

| 方式 | 原理 | Token消耗 | 准确率 |
|------|------|-----------|--------|
| grep搜索 | 文本匹配 | 高 | 低 |
| RAG检索 | 向量相似度 | 中 | 中 |
| **CodeGraph** | **知识图谱** | **低** | **高** |

## 三、核心价值

### A. Token节省81%

**原理**：
- 传统：把整个代码文件喂给AI → 消耗大量Token
- CodeGraph：只提取相关代码片段 → 精准定位，Token极少

**示例**：
```
问题：这个函数被谁调用？

传统方式：
- 读取所有文件（10000+ tokens）

CodeGraph方式：
- 查询知识图谱（500 tokens）
节省95%
```

### B. 本地构建，隐私安全

|------|----------|--------|

## 六、技术栈对接

### 层级映射

| AIX层级 | CodeGraph作用 |
|---------|---------------|
| 第零层（硬件）| AIX Box本地运行 |
| 第一层（记忆）| 代码知识图谱 → Graphify |
| 第二层（工具）| CodeGraph作为Skill封装 |
| 第三层（工作流）| 代码理解 → 知识图谱 → AI调用 |
| 第四层（运行时）| OpenClaw集成 |

### 与现有项目协同

| 项目 | 协同方式 |
|------|----------|
| Graphify | 代码图谱 + 知识图谱 = 全栈理解 |
| Semble | CodeGraph精准定位 + Semble代码搜索 |
| RAG-Anything | 代码 + 文档 + 多模态统一检索 |
| Faster-Whisper | 语音讨论代码 → CodeGraph检索 |

## 三、核心功能模块

### 1. 合约创建

| 功能 | 说明 |
|------|------|
| 合约模板 | 标准化事件合约格式 |
| 事件定义 | 明确的问题、时间范围、判定标准 |
| 初始定价 | 由创建者设定或Kronos建议 |
| 流动性注入 | 创建者注入初始Coin Hour |

**合约创建流程：**
```
用户提出事件 → 审核通过 → 设定参数 → 注入流动性 → 上线交易
```

### 2. 合约交易

| 功能 | 说明 |
|------|------|
| 买入"是" | 认为事件会发生 |
| 买入"否" | 认为事件不会发生 |
| 价格显示 | 当前市场共识概率 |
| 持仓查看 | 我的合约资产 |

**交易界面示例：**
```
合约：那耶村音乐节是否盈利？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
当前价格：35 CH（35%概率）

[买入"是"] [买入"否"]

我的持仓：
  是合约：20 CH
  否合约：10 CH
━━━━━━━━━━━━━━━━━━━━━━━━━━━━
结算时间：2026-06-30
```

### 3. 合约结算

| 结束方式 | 说明 |
|----------|------|
| 自动结算 | 时间到期，结果明确 |
| 人工确认 | 有争议的事件 |
| DAO仲裁 | 争议较大时投票决定 |

**结算流程：**
```
时间到期 → 收集证据 → 确定结果 → 分配收益 → 手续费扣除
```

### 4. 流动性池

| 模型 | 说明 |
|------|------|
| AMM（自动做市商） | 类似Uniswap，x*y=k |
| 订单簿 | 传统撮合模式 |
| 混合模式 | AMM为主，订单簿补充 |

**建议：初期用AMM，简单高效**

## 五、第一批合约建议

## 七、风险管理

### 合约审核

| 审核项 | 标准 |
|--------|------|
| 事件定义 | 明确、无歧义 |
| 时间范围 | 合理、可验证 |
| 判定标准 | 公开、客观 |
| 合规性 | 不涉及敏感话题 |

### 仓位限制

| 限制 | 说明 |
|------|------|
| 单账户仓位上限 | 每个合约最多1000 CH |
| 单合约总仓位 | 每个合约最多10000 CH |
| 创建者限制 | 不能超过初始流动性的50% |

### 争议处理

```
争议提交 → DAO投票 → 结果确认 → 收益分配
投票权重：Token持有量
投票周期：7天
```
